# Project Rules

- Never use Material UI or Bootstrap.
- Always use Tailwind CSS.
- Prefer Server Components when possible.
- Prefer Server Actions for mutations.
- Use TypeScript strict mode.
- Use absolute imports with `@/*`.
- Keep components reusable and avoid duplication.
- Optimize for Vercel deployment.
- Never expose Supabase service role keys to the browser.
- Never ship placeholder-quality UI.
- Every sprint must report updated file structure, database changes, completed features, known limitations, and next todos.
